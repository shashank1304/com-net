OSI Model

The OSI (Open Systems Interconnection) model is a conceptual framework used to understand and implement network protocols in seven layers.

OSI Model Structure

  • Layer 7: Application Layer
  • Layer 6: Presentation Layer
  • Layer 5: Session Layer
  • Layer 4: Transport Layer
  • Layer 3: Network Layer
  • Layer 2: Data Link Layer
  • Layer 1: Physical Layer
  1. Physical Layer: This layer is responsible for the physical connection between devices. It transmits raw bit streams over a physical medium, such as cables or radio waves. It includes hardware components like cables, switches, and network interface cards. Read more
  2. Data Link Layer: This layer provides node-to-node data transfer and handles error detection and correction. It ensures that data is transferred reliably over the physical layer. It includes protocols like Ethernet and PPP, and devices like bridges and switches. Read more
  3. Network Layer: This layer is responsible for routing data packets between devices across different networks. It determines the best path for data to travel from the source to the destination. It includes protocols like IP and ICMP, and devices like routers. Read more
  4. Transport Layer: This layer ensures complete data transfer with error recovery and flow control. It provides reliable data transfer services to the upper layers. It includes protocols like TCP and UDP, which manage data transmission between devices. Read more
  5. Session Layer: This layer manages sessions and controls dialogues between computers. It establishes, maintains, and terminates connections between applications. It includes protocols like NetBIOS and RPC, which manage session establishment and termination. Read more
  6. Presentation Layer: This layer translates data formats and encrypts/decrypts data. It ensures that data is presented in a readable format for the application layer. It includes protocols like SSL/TLS and JPEG, which handle data encoding and encryption. Read more
  7. Application Layer: This layer provides network services to end-users and applications. It includes protocols such as HTTP, FTP, and SMTP that enable communication between software applications. It is the closest layer to the end-user and interacts directly with software applications. Read more
